# 决策记忆闭环

这个 skill 只做一件事：

> 在反复出现的决策问题里，先恢复已有结论，避免重复纠结，再把用户确认过的新结论写回记忆。

它不是万能人生数据库，也不是情绪日记、知识库或复杂周报系统。

## 核心流程

1. 判断当前问题是否延续历史决策主题。
2. 先读取短期记忆。
3. 再读取当前决策看板。
4. 只读取相关长期项目记忆。
5. 先回顾已有结论，再回答当前问题。
6. 如果用户确认了新的稳定结论，再更新记忆或看板。

## 记忆位置

默认使用当前工作区下的中文目录：

```text
记忆/
```

如果不存在，可以回退到英文目录：

```text
memory/
```

推荐结构：

```text
记忆/
  短期记忆.md
  长期记忆/
    当前决策看板.md
    项目列表.md
    项目列表/
      travel_plan.md
      side_project.md
      career_plan.md
```

英文目录也可以：

```text
memory/
  short-term-memory.md
  long-term-memory/
    decision-board.md
    project-index.md
    projects/
```

## 读取顺序

1. 读取短期记忆。
2. 读取当前决策看板。
3. 识别当前主题。
4. 只读取当前主题相关的长期项目文件。
5. 默认不要读取全部长期记忆。

## 什么时候触发

适合触发：

- 方向选择。
- 项目继续、暂停、放弃。
- 当前优先级排序。
- 下一步行动判断。
- 反复出现的旧问题。
- 用户要求记录、更新或删除长期记忆。
- 用户要求增加、删除、调整当前决策看板内容。

不适合触发：

- 普通技术排错。
- 一次性事实查询。
- 翻译、改写、润色。
- 与历史决策无关的闲聊。

## 当前决策看板

当前决策看板用于回答四个问题：

1. 当前最重要的 1~3 件事是什么？
2. 哪些结论已经定了，不要反复重启？
3. 哪些事项暂缓或不再作为当前重点？
4. 最近什么时候需要复盘？

看板只保存当前阶段的决策入口，不保存完整项目细节。

看板模板：

```md
# 当前决策看板

## 当前最重要的 3 件事

暂无

## 已定结论

暂无

## 暂缓或不再反复讨论

暂无

## 最近复盘点

暂无

## log

暂无
```

## 看板维护规则

- 只有用户明确要求调整看板，或本轮已经确认当前优先级变化时，才更新看板。
- `当前最重要的 3 件事` 最多保留 3 条。
- 新增重点时，必要时移除或降级旧重点。
- `已定结论` 只放跨项目、会影响后续决策的稳定结论。
- 项目内细节写入对应长期项目文件，不要塞进看板。
- `暂缓或不再反复讨论` 用于保存当前不应继续消耗注意力的事项。
- `最近复盘点` 只放明确时间或明确触发条件。
- 看板变化要在 `log` 里追加一句简短记录。

删除或降级规则：

- 用户明确说“去掉”“不关注了”“暂缓”“不再作为重点”时，从当前重点移除。
- 必要时把移除项放入 `暂缓或不再反复讨论`。
- 某事项已经进入对应长期项目、不再需要占用当前注意力时，可以从看板移除。
- 复盘点过期或完成后，更新为新的复盘点。
- 不要因为一次临时兴趣就改看板。

## 长期项目记忆

推荐结构：

```md
# 项目名称

## summary

## status

## plan

## decision criteria

## question

## warning

## todo

## log

## note
```

不要为了模板完整而脑补内容。没有内容就写“暂无”或留空。

## 写回规则

只写入：

- 用户明确表达的内容。
- 用户确认过的结论。
- 本轮讨论中已经清楚形成的稳定判断。

不要写入：

- 临时情绪。
- 寒暄。
- 未确认的 AI 建议。
- AI 自己脑补的计划、风险、待办。
- 为了填满模板而扩写的低价值内容。

当用户说“ok，先这样”“就先这样”“先按这个”“可以，就这样”等，视为当前稳定结论已确认，但仍然只写确认过的内容。

## 默认回答方式

可以按这个顺序回答：

```md
## 已有上下文

简要恢复当前决策看板、相关历史结论、提醒和未解决问题。

## 当前回答

直接给判断、理由、代价和下一步。

## 记忆更新

说明是否需要新增、修改或删除记忆，是否需要同步当前决策看板。
```

如果问题很小，可以不用显式标题，保持简短。

## 第一版目标

第一版只追求一个结果：

> 同一个问题第二次出现时，不要像第一次见到一样回答。

自动化、同步、图形界面、云端汇总都可以放到以后。
