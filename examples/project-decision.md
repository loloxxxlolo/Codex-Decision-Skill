# Example: Project Decision

## User Question

Should I keep building this side project, or stop for now?

## Existing Context

- The project has a working v1.
- The next feature list is growing.
- The user prefers small loops over large architecture.

## Current Answer

Keep the project, but only upgrade features that strengthen the core loop.

Decision criteria:

Continue:
- the user still uses it personally,
- v1 is stable,
- the next feature clearly reduces friction.

Pause:
- usage drops,
- maintenance becomes heavier than value,
- new features make the product slower or more complex.

Upgrade:
- the feature directly improves the core use case,
- the feature increases review value,
- the feature reduces data-loss risk.

## Memory Update

Add these criteria to the project memory under `decision criteria`.
